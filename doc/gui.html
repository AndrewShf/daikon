Introduction

By default, Daikon outputs a long textual list of invariants. Sometimes one is overwhelmed by the sheer number of invariants. The graphical user interface (GUI) is designed to make the task of browsing invariants easier. The GUI contains a tree which hierarchically organizes program points according to their class and method. Using the GUI, you can look at invariants for only the methods and program points you care about. You can also sort invariants by a property (eg, probability) or filter invariants (eg, only look at invariants containing a certain variable).

Starting up the GUI

To use the GUI, first make a .inv file by invoking Daikon with the -o option; to create foo.inv, you would do 
    java daikon.Daikon -o foo.inv decl-files... trace-files...
Then, run the GUI with the .inv file as an argument: 
    java daikon.gui.InvariantsGUI foo.inv
You can also run the GUI with no arguments. If you do this, you will immediately see a file browser that asks you to choose a .inv file.

Using the GUI

This GUI consists of two windows. The main window is the Invariants Display, where you can navigate through program points and view invariants. The other window is the Filter Control Panel, where you can control the filters that are being applied to the Invariants Display.

Invariants Display

File menu
The Load file menu option brings up a file browser which you can use to select another .inv file. The Quit menu options lets you quit the GUI.
Program point tree
The top part of the Invariants Display consists of the Program point tree - a hierarchy of program points broken down by class and method. Click on various program points in the tree to see tables of invariants for those program points. You may use the Ctrl and Shift keys to make an arbitrary selection of program points. Selecting a class or method will automatically select all the Ppt's associated with that method or class.

Invariant tables
At the bottom of the Invariants Display, we have Invariant tables which display the actual invariants. Here are some tricks for navigating through the tables:
  • Use Alt along with the up or down arrow keys to scroll a table at a time.
  • Sort a column in a table by clicking on the column header. Shift-click to sort the column in reverse order.

Control Panel

The Control Panel gives you control over which invariants are displayed.

Property filters
By default, all the Property filters are on. Property filters narrow down the list of invariants that are displayed. For example, the "Suppress unjustified invariants" filter hides unjustified invariants, which most people will not care about. You can adjust which Property filters are being applied.

Variable filters
Variable filters let you display only the invariants that contain certain variables. The left-hand box displays the variables that you are interested. By default, the box is empty and invariants containing any variables are displayed. There are two ways to add variables to the variables list:

  1. Type the name of the variable into the Add variable textfield, and click the Add variable button (or just press enter).
  2. Each Invariant table in the Invariants Display has a Show variables button. Click on that button to see a list of variables for that program point. Select any set of variables and click the Filter on selected variables button.

To remove variables from the variables list, select the variables in question and click on the Remove selected variables button.

If you click on the any variable checkbox, invariants will be displayed if they contain any of the variables in the variables list. If you click on the all variables checkbox, invariants will be displayed if they contain all of the variables in the variables list.
